# أقم — Aqim

مدرب شخصي لبناء عادة الصلاة تدريجيًا، أسبوعًا بعد أسبوع، بدل تطبيق مواقيت عادي.

هذا مشروع Flutter كامل (Android) جاهز للتجريب على هاتف حقيقي. النسخة الحالية
تستعمل بيانات محلية فقط (SharedPreferences) — لا يوجد خادم أو حساب مستخدم بعد.

## أسرع طريقة للتجريب: عبر GitHub Actions (بدون تثبيت Flutter)

1. أنشئ مستودعًا جديدًا على GitHub وارفع هذا المجلد إليه.
2. بمجرد الدفع (push) إلى فرع `main`، سيعمل الووركفلو `Build Aqim APK`
   تلقائيًا (Actions → Build Aqim APK).
3. بعد انتهاء التشغيل بنجاح، افتح تبويب Artifacts وحمّل `aqim-debug-apk`.
4. انقل ملف `app-debug.apk` إلى هاتفك (عبر الإيميل، Google Drive، أو كابل
   USB) وثبّته يدويًا — قد تحتاج للسماح بـ"تثبيت من مصادر غير معروفة" في
   إعدادات هاتفك.

## التجريب محليًا (إن كان Flutter مثبتًا على جهازك)

```bash
flutter pub get
flutter run          # يوصل الهاتف عبر USB مع تفعيل خيارات المطور + تصحيح USB
# أو لبناء ملف APK مباشرة:
flutter build apk --debug
# الملف الناتج: build/app/outputs/flutter-apk/app-debug.apk
```

## بنية المشروع

```

  ads/                       # معرّفات AdMob + بانر إعلاني قابل لإعادة الاستخدام
  theme/app_theme.dart       # ألوان وخطوط أقم (نيلي الليل، ذهبي الفجر)
  models/prayer.dart         # الصلوات الخمس، الآيات/الأحاديث، أسباب الفوات
  models/adhkar.dart         # أذكار ما قبل وما بعد الصلاة كاملة

  services/location_service.dart      # صلاحية الموقع + إحداثيات الهاتف
  services/prayer_times_service.dart  # جلب أوقات الصلاة الحقيقية (AlAdhan API)
  services/notification_service.dart  # تذكيرات قبل/بعد كل صلاة

  widgets/day_arc.dart       # عنصر "قوس اليوم" المميز (CustomPainter)
  screens/

    home_screen.dart         # قوس اليوم + الصلاة القادمة + رسالة الهوية
    pre_prayer_screen.dart   # آية / حديث / فائدة + رابط أذكار ما قبل الصلاة
    dhikr_screen.dart        # عداد الذكر السريع (٣٣/٣٣/٣٤) + رابط الأذكار الكاملة
    adhkar_flow_screen.dart  # شاشة عامة تعرض أي قائمة أذكار خطوة بخطوة
    identity_screen.dart     # تعزيز الهوية بعد إتمام الصلاة
    reason_screen.dart       # "لماذا لم أُصلِّ" + اقتراح مخصص
    week_report_screen.dart  # لوحة الحياة الأسبوعية
```

## الإعلانات (AdMob)

التطبيق مربوط بـ Google Mobile Ads SDK وفيه بانر إعلاني في شاشتين فقط:
**الرئيسية** و**التقرير الأسبوعي** — بشكل مقصود لا يوجد أي إعلان في شاشات
الصلاة أو الأذكار أو الهوية، احترامًا لطبيعة المحتوى.

**الوضع الحالي**: يستعمل معرّفات اختبار رسمية من Google (`ca-app-pub-3940256099942544/...`)
— تعمل فورًا بلا أي إعداد إضافي، لكن بلا أي عائد مالي حقيقي.

**قبل النشر على المتجر، خاصك:**
1. أنشئ حسابًا على [AdMob](https://admob.google.com) وسجّل تطبيق أقم فيه.
2. استبدل معرّف التطبيق فـ
   `android/app/src/main/AndroidManifest.xml` (بحث عن `APPLICATION_ID`)
   بمعرّف تطبيقك الحقيقي.
3. استبدل معرّف البانر فـ `lib/ads/ad_ids.dart` بمعرّف وحدة الإعلان
   الحقيقي.
4. انتظر مراجعة Google للتطبيق (عادة تستغرق ساعات لأول مرة) قبل أن تظهر
   إعلانات حقيقية بدل رسالة "لا يوجد إعلان متاح".

## مواقيت الصلاة الحقيقية

التطبيق يجلب الآن أوقات الصلاة الفعلية بدل الأوقات الوهمية:
- يطلب صلاحية الموقع عند أول تشغيل (يحتاجها لحساب الأوقات بدقة).
- يستعمل [AlAdhan API](https://aladhan.com) بطريقة حساب وزارة الأوقاف
  والشؤون الإسلامية المغربية (method=21).
- يعمل في الخلفية بلا حجب شاشة الإقلاع — الأوقات تظهر وتتحدّث في "قوس
  اليوم" وبطاقة "الصلاة القادمة" بمجرد توفّرها.
- **يتدهور بأمان (graceful fallback)**: إن رُفضت صلاحية الموقع أو تعذّر
  الاتصال بالإنترنت، يعرض التطبيق الأوقات الاحتياطية الثابتة بدل أن
  يتعطّل أو يظهر فارغًا.
- آخر إحداثيات معروفة تُحفَظ محليًا (SharedPreferences) وتُستعمل كبديل
  إن فشل تحديد الموقع في مرة لاحقة لكنه نجح سابقًا.

**ملاحظة**: منطق التقدّم عبر الصلوات (أيّها "القادمة" الآن) يبقى تسلسليًا
بحسب ما أكمله المستخدم فعليًا، وليس مرتبطًا بالساعة الفعلية — الأوقات
الحقيقية هنا للعرض والتذكير فقط. هذا خيار تصميم مقصود يسمح للمستخدم
بمتابعة صلواته بترتيبها الطبيعي حتى لو فتح التطبيق متأخرًا عن وقتها.

## تذكيرات الصلاة (إشعارات حقيقية)

التطبيق يرسل الآن إشعارين لكل صلاة من الصلوات الخمس، بشرط توفّر الأوقات
الحقيقية (لا يُجدوَل أي إشعار على الأوقات الاحتياطية الوهمية):

1. **قبل الصلاة بعشر دقائق**: "اقترب وقت [الصلاة] — استعد لصلاة [الصلاة]".
   الضغط عليه يفتح شاشة الآية/الحديث/الفائدة لتلك الصلاة مباشرة.
2. **بعد الصلاة بعشرين دقيقة**: "هل صليت [الصلاة]؟" — الضغط عليه يفتح
   نفس الشاشة، وفيها زرّا "صليت" و"لم أُصلِّ بعد" لتسجيل الحالة فورًا.

**ملاحظات تقنية مهمة:**
- تُعاد جدولة الإشعارات كل مرة تُجلَب فيها أوقات جديدة بنجاح (عادة مرة
  عند فتح التطبيق يوميًا). فتح التطبيق يوميًا ضروري حاليًا لتحديث
  الإشعارات لليوم الجديد — لا توجد آلية لإعادة الجدولة تلقائيًا في
  الخلفية أو بعد إعادة تشغيل الهاتف بعد.
- تُستعمل جدولة "غير دقيقة" (`inexactAllowWhileIdle`) عمدًا، فقد تتأخر
  الإشعارات بضع دقائق أحيانًا (خصوصًا فـ وضع توفير الطاقة). هذا اختيار
  مقصود لتفادي طلب صلاحية "المنبّهات الدقيقة" الحساسة على أندرويد 12+
  (`SCHEDULE_EXACT_ALARM`)، والتي قد تُعقّد مراجعة Google Play لاحقًا.
- يحتاج Android 13+ صلاحية إشعارات صريحة — يطلبها التطبيق تلقائيًا عند
  أول تشغيل.
- تتطلب `flutter_local_notifications` تفعيل *core library desugaring*
  فـ `android/app/build.gradle` (مُفعَّل مسبقًا فـ هذا المشروع).

## ما تم تبسيطه في هذه النسخة (ولخطوات لاحقة)

- **إعادة الجدولة التلقائية**: الإشعارات تُجدوَل عند فتح التطبيق فقط.
  لجعلها تُحدَّث تلقائيًا كل يوم بلا فتح التطبيق، ستحتاج مهمة خلفية
  (WorkManager) — خطوة إضافية غير مُنفَّذة حاليًا.
- **الصوت**: أزرار 🔊 موجودة على كل ذكر وتعمل فعليًا (عبر `audioplayers`)،
  لكنها بلا تسجيلات حتى الآن — راجع قسم "إضافة التسجيلات الصوتية" أعلاه.
- **التوقيع للنشر على المتجر**: الإصدار الحالي (`--release`) موقّع بمفتاح
  Debug مؤقتًا كي يُبنى بدون إعداد إضافي. قبل رفعه إلى Play Store، أنشئ
  keystore حقيقي واربطه عبر GitHub Secrets (نفس الأسلوب المستعمل في "سُفرة").
- **الخطوط**: تُحمَّل عبر `google_fonts` عند أول تشغيل (تتطلب إنترنت مرة
  واحدة، ثم تُخزَّن محليًا).
