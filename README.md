# أقم — Aqim

مدرب شخصي لبناء عادة الصلاة تدريجيًا، أسبوعًا بعد أسبوع، بدل تطبيق مواقيت عادي.

هذا مشروع Flutter كامل (Android) جاهز للتجريب على هاتف حقيقي. النسخة الحالية
تستعمل بيانات محلية فقط (SharedPreferences) — لا يوجد خادم أو حساب مستخدم بعد.

## أسرع طريقة للتجريب: عبر GitHub Actions (بدون تثبيت Flutter)

1. أنشئ مستودعًا جديدًا على GitHub وارفع هذا المجلد إليه.
2. بمجرد الدفع (push) إلى فرع `main`، سيعمل الووركفلو `Build Aqim APK`
   تلقائيًا (Actions → Build Aqim APK).
3. بعد انتهاء التشغيل بنجاح، افتح تبويب Artifacts وحمّل `aqim-debug-apk`.
4. انقل ملف `app-debug.apk` إلى هاتفك (عبر الإيميل، Google Drive، أو كابل
   USB) وثبّته يدويًا — قد تحتاج للسماح بـ"تثبيت من مصادر غير معروفة" في
   إعدادات هاتفك.

## التجريب محليًا (إن كان Flutter مثبتًا على جهازك)

```bash
flutter pub get
flutter run          # يوصل الهاتف عبر USB مع تفعيل خيارات المطور + تصحيح USB
# أو لبناء ملف APK مباشرة:
flutter build apk --debug
# الملف الناتج: build/app/outputs/flutter-apk/app-debug.apk
```

## الإعلانات (AdMob)

التطبيق مربوط بـ Google Mobile Ads SDK وفيه بانر إعلاني في شاشتين فقط:
**الرئيسية** و**التقرير الأسبوعي** — بشكل مقصود لا يوجد أي إعلان في شاشات
الصلاة أو الأذكار أو الهوية، احترامًا لطبيعة المحتوى.

**الوضع الحالي**: يستعمل معرّفات اختبار رسمية من Google (`ca-app-pub-3940256099942544/...`)
— تعمل فورًا بلا أي إعداد إضافي، لكن بلا أي عائد مالي حقيقي.

**قبل النشر على المتجر، خاصك:**
1. أنشئ حسابًا على [AdMob](https://admob.google.com) وسجّل تطبيق أقم فيه.
2. استبدل معرّف التطبيق فـ
   `android/app/src/main/AndroidManifest.xml` (بحث عن `APPLICATION_ID`)
   بمعرّف تطبيقك الحقيقي.
3. استبدل معرّف البانر فـ `lib/ads/ad_ids.dart` بمعرّف وحدة الإعلان
   الحقيقي.
4. انتظر مراجعة Google للتطبيق (عادة تستغرق ساعات لأول مرة) قبل أن تظهر
   إعلانات حقيقية بدل رسالة "لا يوجد إعلان متاح".

## مواقيت الصلاة الحقيقية

التطبيق يجلب الآن أوقات الصلاة الفعلية بدل الأوقات الوهمية:
- يطلب صلاحية الموقع عند أول تشغيل (يحتاجها لحساب الأوقات بدقة).
- يستعمل [AlAdhan API](https://aladhan.com) بطريقة حساب وزارة الأوقاف
  والشؤون الإسلامية المغربية (method=21).
- يعمل في الخلفية بلا حجب شاشة الإقلاع — الأوقات تظهر وتتحدّث في "قوس
  اليوم" وبطاقة "الصلاة القادمة" بمجرد توفّرها.
- **يتدهور بأمان (graceful fallback)**: إن رُفضت صلاحية الموقع أو تعذّر
  الاتصال بالإنترنت، يعرض التطبيق الأوقات الاحتياطية الثابتة بدل أن
  يتعطّل أو يظهر فارغًا.
- آخر إحداثيات معروفة تُحفَظ محليًا (SharedPreferences) وتُستعمل كبديل
  إن فشل تحديد الموقع في مرة لاحقة لكنه نجح سابقًا.

**ملاحظة**: منطق التقدّم عبر الصلوات (أيّها "القادمة" الآن) يبقى تسلسليًا
بحسب ما أكمله المستخدم فعليًا، وليس مرتبطًا بالساعة الفعلية — الأوقات
الحقيقية هنا للعرض والتذكير فقط. هذا خيار تصميم مقصود يسمح للمستخدم
بمتابعة صلواته بترتيبها الطبيعي حتى لو فتح التطبيق متأخرًا عن وقتها.

## تذكيرات الصلاة (إشعارات حقيقية)

التطبيق يرسل **ثلاثة** تنبيهات لكل صلاة من الصلوات الخمس، بشرط توفّر
الأوقات الحقيقية (لا يُجدوَل أي إشعار على الأوقات الاحتياطية الوهمية):

1. **منبّه الاستعداد** (10 دقائق قبل الوقت، قابل للتخصيص): "استعد لصلاة
   [الصلاة]" — منبّه حقيقي بصوت مخصّص، يعمل حتى والهاتف مقفل أو صامت
   (شاشة كاملة + فئة "منبّه" فـ أندرويد)، لمساعدتك على النهوض والاستعداد
   — خصوصًا للفجر. الضغط عليه يفتح شاشة الآية/الحديث/الفائدة.
2. **الأذان**: عند دخول وقت الصلاة بالضبط، إشعار بصوت الأذان (إن أضفته
   — راجع "إضافة أصوات المنبّه والأذان" أدناه).
3. **هل صليت؟** (20 دقيقة بعد الوقت، قابل للتخصيص): بزر إجراء **"أجّل 15
   دقيقة"** يُعيد جدولة نفس الإشعار بلا فتح التطبيق.
4. **ملخّص أسبوعي**: كل جمعة الساعة ٨ مساءً، "أتممت X من ٣٥ صلاة هذا
   الأسبوع 🌙".

**تخصيص التوقيت**: من شاشة الإعدادات (أيقونة ⚙️ في الشاشة الرئيسية) يقدر
المستخدم يختار توقيت منبّه الاستعداد (5 إلى 30 دقيقة) وتذكير "هل صليت؟"
(10 إلى 45 دقيقة) — يُعاد جدولة كل الإشعارات فورًا عند التغيير.

**استثناء توفير البطارية**: عند أول فتح للتطبيق، يتحقق تلقائيًا إن كان
الهاتف يستثني التطبيق من توفير الطاقة. إن لم يكن كذلك، يعرض حوارًا
يوجّه المستخدم لصفحة الإعدادات المناسبة (تشمل إعدادات خاصة بالشركة
المصنّعة على أجهزة Xiaomi/Huawei/Oppo وغيرها، وهي غالبًا السبب الرئيسي
لتأخّر أو اختفاء التذكيرات على هذا النوع من الهواتف). يمكن إعادة فتح
هذه الإعدادات لاحقًا من شاشة الإعدادات فـ أي وقت.

### إضافة أصوات المنبّه والأذان

**مهم: هذه الأصوات لا تُضاف فـ `assets/audio/` كباقي التسجيلات** —
أصوات الإشعارات على أندرويد تحتاج مكانًا خاصًا داخل المشروع الأصلي:

```
android/app/src/main/res/raw/wakeup_alarm.mp3   ← صوت منبّه الاستعداد
android/app/src/main/res/raw/adhan.mp3          ← صوت الأذان
```

ملاحظات مهمة عن هذين الملفين تحديدًا:
- اسم الملف يجب أن يكون بأحرف إنجليزية صغيرة وأرقام وشرطة سفلية `_` فقط
  (بدون مسافات أو شرطات `-`) — قيد من أندرويد نفسه.
- لـ `adhan.mp3`: يُفضَّل مقطع قصير (٢٠-٤٠ ثانية) بدل الأذان الكامل،
  لضمان تشغيله بشكل موثوق كصوت إشعار على مختلف الأجهزة.
- **تنبيه حقوق النشر**: إن استعملت تسجيل أذان لمؤذّن معروف (كالحرم
  المكي مثلاً)، تأكد من أن الترخيص يسمح بإعادة الاستعمال داخل تطبيق،
  أو سجّل أذانًا خاصًا بك — الأصوات المتداولة على الإنترنت ليست كلها
  حرة الاستعمال.
- إن لم تُضَف الملفات، يعمل كل شيء بلا كراش — تُستعمل نغمة الإشعار
  الافتراضية للنظام بدلاً منها.
- بعد إضافة الملفين، لا حاجة لـ `flutter pub get` (لأنهما موارد أندرويد
  أصلية وليست Flutter assets) — أعد البناء مباشرة.
- **حالة هذا المشروع الآن**: الملفان مُضافان فعلًا (`wakeup_alarm.mp3`
  ~34 ثانية، و`adhan.mp3` ~4 دقائق كاملة). الأذان الكامل قد يُقطَع أو
  يتصرّف بشكل مختلف على بعض الأجهزة كصوت إشعار طويل نسبيًا — إن لاحظت
  ذلك أثناء التجريب، جرّب استبداله بمقطع أقصر (٢٠-٤٠ ثانية) من نفس
  التسجيل.

## أقرب مسجد

زر "أقرب مسجد" فـ الشاشة الرئيسية يعرض قائمة بأقرب المساجد (ضمن 3 كم)
مرتّبة حسب المسافة، مع زر "الاتجاهات" يفتح خرائط Google للتوجّه إليها.

- يستعمل بيانات **OpenStreetMap** عبر Overpass API — مصدر مفتوح ومجاني
  بلا حاجة لمفتاح API أو فوترة (بخلاف Google Places API).
- **قيد معروف**: تغطية OpenStreetMap للمساجد تعتمد على مساهمات
  المستخدمين، فقد تكون غير كاملة فـ بعض المناطق (خصوصًا القرى الصغيرة).
  المساجد المسجَّلة فـ المدن الكبرى فـ المغرب عادة موثّقة بشكل جيد.
- يستعمل آخر إحداثيات معروفة (نفس المصدر المستعمل لأوقات الصلاة) إن
  تعذّر تحديد الموقع الحالي فورًا.

### ملاحظات تقنية عن الإشعارات
- تُعاد جدولة كل الإشعارات (منبّه الاستعداد + الأذان + الملخّص الأسبوعي)
  كل مرة تُجلَب فيها أوقات جديدة بنجاح (عادة مرة عند فتح التطبيق يوميًا).
  فتح التطبيق يوميًا ضروري حاليًا لتحديث الإشعارات لليوم الجديد — لا
  توجد آلية لإعادة الجدولة تلقائيًا في الخلفية أو بعد إعادة تشغيل
  الهاتف بعد.
- تُستعمل جدولة "غير دقيقة" (`inexactAllowWhileIdle`) عمدًا، فقد تتأخر
  الإشعارات بضع دقائق أحيانًا (خصوصًا فـ وضع توفير الطاقة). هذا اختيار
  مقصود لتفادي طلب صلاحية "المنبّهات الدقيقة" الحساسة على أندرويد 12+
  (`SCHEDULE_EXACT_ALARM`)، والتي قد تُعقّد مراجعة Google Play لاحقًا.
- زر "أجّل 15 دقيقة" لا يعمل إن كان النظام قد أغلق التطبيق بالكامل من
  الذاكرة (وليس فقط فـ الخلفية) — يحتاج ذلك معالجة إضافية عبر background
  isolate غير مُنفَّذة حاليًا؛ الضغط عليه فـ تلك الحالة سيفتح التطبيق
  بدل تأجيل الإشعار بصمت.
- يحتاج Android 13+ صلاحية إشعارات صريحة — يطلبها التطبيق تلقائيًا عند
  أول تشغيل.
- تتطلب `flutter_local_notifications` تفعيل *core library desugaring*
  فـ `android/app/build.gradle` (مُفعَّل مسبقًا فـ هذا المشروع).
- **صلاحية `USE_FULL_SCREEN_INTENT`** (لمنبّه الاستعداد): على أندرويد
  14+، قد يرفض Google Play منح هذه الصلاحية تلقائيًا للتطبيقات إلا إن
  صرّحت بامتلاكها وظيفة "منبّه" حقيقية عند النشر على المتجر — راجع
  [توثيق Google](https://source.android.com/docs/core/permissions/fsi-limits)
  قبل النشر. لا يؤثر هذا على التجريب المحلي أو عبر GitHub Actions.

## ما تم تبسيطه في هذه النسخة (ولخطوات لاحقة)

- **إعادة الجدولة التلقائية**: الإشعارات تُجدوَل عند فتح التطبيق فقط.
  لجعلها تُحدَّث تلقائيًا كل يوم بلا فتح التطبيق، ستحتاج مهمة خلفية
  (WorkManager) — خطوة إضافية غير مُنفَّذة حاليًا.
- **الصوت**: أزرار 🔊 موجودة على كل ذكر وتعمل فعليًا (عبر `audioplayers`)،
  لكنها بلا تسجيلات حتى الآن — راجع قسم "إضافة التسجيلات الصوتية" أعلاه.
- **التوقيع للنشر على المتجر**: الإصدار الحالي (`--release`) موقّع بمفتاح
  Debug مؤقتًا كي يُبنى بدون إعداد إضافي. قبل رفعه إلى Play Store، أنشئ
  keystore حقيقي واربطه عبر GitHub Secrets (نفس الأسلوب المستعمل في "سُفرة").
- **الخطوط**: تُحمَّل عبر `google_fonts` عند أول تشغيل (تتطلب إنترنت مرة
  واحدة، ثم تُخزَّن محليًا).
